
Designova Setup Guide

This guide explains how to run the project from scratch on a new laptop.

It covers:
- software to install
- MySQL database setup
- backend setup
- frontend setup
- admin login setup
- commands to run the project
- common errors and fixes

1. Project Overview

This project currently runs as two separate apps:

1. frontend
   - React + TypeScript + Vite
   - runs the user interface
   - default frontend URL: http://localhost:5173

2. interior-design-nextjs
   - Next.js app used as the backend API server
   - handles authentication, CRUD APIs, and MySQL connection
   - backend API base URL: http://localhost:3000/api

Database schema file:
- interior_design/database/schema.sql

Important note:
- The frontend is already configured to call the backend at http://localhost:3000/api
- CORS is already configured to allow http://localhost:5173
- so the easiest setup is:
  - backend on port 3000
  - frontend on port 5173

5. Database Setup

This step is required before running the backend.

5.1 Create the Database

Open MySQL Workbench or MySQL command line and run:

CREATE DATABASE interior_design_db;

Then select it:

USE interior_design_db;

5.2 Import the Schema

You have two ways to import the schema.

Option A: Using MySQL Workbench
1. Open MySQL Workbench
2. Connect to your local MySQL server
3. Open the file:
   major_project/interior_design/database/schema.sql
4. Run the full script

Option B: Using MySQL command line
From PowerShell:

mysql -u root -p interior_design_db < "C:\path\to\major_project\interior_design\database\schema.sql"

Replace C:\path\to\major_project with the actual path on your friend's laptop.

5.3 Tables Created by the Schema

6. Backend Setup

The backend is in:
- interior-design-nextjs

6.1 Open Terminal in Backend Folder
cd C:\path\to\major_project\interior-design-nextjs

6.2 Install Dependencies
npm install

6.3 Create the Backend Environment File

Inside interior-design-nextjs, create a file named:
- .env.local

Add this content:

DB_HOST=localhost
DB_USER=root
DB_PASSWORD=YOUR_MYSQL_PASSWORD
DB_NAME=interior_design_db
JWT_SECRET=change-this-to-a-long-random-secret

Replace:
- YOUR_MYSQL_PASSWORD with the MySQL password on your friend's laptop
- JWT_SECRET with any strong random secret

Important:
- use the same database name created earlier: interior_design_db
- do not keep the old password from another machine

6.4 Start the Backend

Run the backend explicitly on port 3000:
npm run dev -- --port 3000

Expected result:
- backend starts successfully
- Next.js server runs on http://localhost:3000

6.5 Backend Test URLs

Open these in the browser after backend startup:
- http://localhost:3000
- http://localhost:3000/api/projects

If the backend is working, the API route should return JSON.

7. Frontend Setup

The frontend is in:
- frontend

7.1 Open a Second Terminal
Open a new terminal window and run:
cd C:\path\to\major_project\frontend

7.2 Install Dependencies
npm install

7.3 Create the Frontend Environment File

Inside frontend, create or verify the file:
- .env

Its content should be:

This tells the frontend to connect to the local backend.

7.4 Start the Frontend

Run:

Expected result:

- frontend runs at http://localhost:5173

12. Common Problems and Fixes

Problem 1: Frontend loads but API calls fail

Possible reasons:
- backend is not running
- backend is running on the wrong port
- frontend .env has the wrong API base URL

Fix:
1. make sure backend is running on 3000
2. verify:
VITE_API_BASE_URL=http://localhost:3000/api

Problem 2: CORS error in browser console

The backend currently allows:
- http://localhost:5173

If the frontend runs on a different port, update:
- interior-design-nextjs/lib/cors.ts

Current allowed dev origin:
"http://localhost:5173"

If needed, change it and restart the backend.

Problem 3: MySQL connection error

Possible reasons:
- wrong MySQL username
- wrong MySQL password
- database not created
- .env.local is missing or wrong

Fix:
Check the backend env file:

DB_HOST=localhost
DB_USER=root
DB_PASSWORD=YOUR_MYSQL_PASSWORD
DB_NAME=interior_design_db
JWT_SECRET=change-this-to-a-long-random-secret

Problem 4: Login fails even though the site opens

Possible reasons:
- admin user was not inserted
- inserted password hash is wrong
- user role is not admin

Fix:
Run:
SELECT user_id, name, email, role FROM users WHERE email = 'admin@interiordesign.com';

The role must be admin.

Problem 5: Frontend builds but backend APIs still fail

Check:
- database is imported correctly
- all schema tables exist
- backend terminal shows no DB error

Run this in MySQL:
SHOW TABLES;

Expected tables:
- users
- projects
- rooms
- design_images
- enquiries
- testimonials

15. Final Notes

- Keep both backend and frontend terminals running during development
- Do not change the backend port unless you also update the frontend API URL
- Do not skip the admin insert query
- Use MySQL 8 if possible because the schema uses MySQL 8 compatible defaults
- If you share this project again, do not share your own real database password in .env files
